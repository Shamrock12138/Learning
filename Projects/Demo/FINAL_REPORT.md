# 无人机充电系统强化学习项目 - 最终总结

## 项目目标
开发一个强化学习系统，让充电无人机能够自主为任务无人机充电，同时在基站为自身充电，从而最大化任务无人机的生存时间。

## 解决的关键问题

### 1. 初期问题
- 充电无人机无法有效追踪任务无人机
- 任务无人机存活率极低（接近0%）
- 充电行为几乎不发生
- 训练不稳定，奖励函数设计不合理

### 2. 解决方案演进

#### 第一阶段：环境修复
- 修复了充电逻辑中的bug（充电距离判断、状态传递等）
- 优化了奖励函数结构，增加对充电行为的激励

#### 第二阶段：状态表示改进
- 从简单状态扩展到增强状态（13维）
- 引入相对位置、距离、角度等关键信息

#### 第三阶段：混合策略方法
- 开发了基于行为树的混合策略
- 初期使用规则指导，后期转向纯学习
- 平衡了探索与利用的关系

#### 第四阶段：深度训练
- 增加训练轮数至1000轮
- 实现了收敛性分析
- 优化了学习率和探索策略

## 核心技术特点

### 1. 行为状态设计
```python
behavior_state = [
    # 自身信息
    cuav_pos_x, cuav_pos_y, cuav_battery,
    # 任务机信息  
    tuav_pos_x, tuav_pos_y, tuav_battery,
    # 基站信息
    base_pos_x, base_pos_y,
    # 相对信息
    rel_tuav_dx, rel_tuav_dy,  # 相对任务机位置
    rel_base_dx, rel_base_dy,  # 相对基站位置
    manhattan_dist_to_task     # 曼哈顿距离
]
```

### 2. 奖励函数优化
- 充电成功奖励：+15.0
- 低电量任务机接近奖励：随距离递减
- 基站充电奖励：鼓励自我维护
- 任务机坠毁惩罚：-100.0（严厉惩罚）

### 3. 混合策略逻辑
```
if (can_charge_task && tuav_battery < 0.5):
    action = CHARGE_TASK
elif (cuav_battery < 0.2):
    action = NAVIGATE_TO_BASE
else:
    action = NAVIGATE_TO_TASK
```
